RESTful API with RDBMS

Intent

To create an application that uses:
  • Spring-Boot Plugin to help facilitate a MVC framework
  • Hibernate for model to table db mapping
  • GitHub to push application files
  • Maven to build and run the application
  • Docker to package the app into a single platform independent image

Usage

  • Jetty

    To quickly execute the application, issue the following from the app root - mvn spring-boot:run

  • Docker

    Build: docker build -t namespace/repository:version .
    namespace - your_company_name
    repository - your_app_name
    version - latest

    The '.' at the end of the build command indicates to use the Dockerfile and content of the current directory

    Run:The following usage of docker run assumes MySQL is in use by the app and is listening on port 3306

    docker run -it -d -p 52000:3306 -p 8080:8080 /:

    52000 is used by way of example for the purpose of avoiding a port conflict in the event MySQL is being run locally on the host machine.

    Note:
    This package does not contain UI templates, making it head-less and a server-based RESTful implementation. Accessing the application in a browser will reveal a test harness to allow running through the high level CRUD methods of each entity of the model.
