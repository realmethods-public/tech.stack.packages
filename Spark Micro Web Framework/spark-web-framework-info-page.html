Spark Micro Web Framework

Intent

To create an application that uses:

  • Spark Java Micro Web Framework to faciltate the view/controller of an MVC framework
  • Fully functional front-end using Javascript, HTML5, JQuery, CSS, and Bootstrap
  • Hibernate for object to table db mapping
  • GitHub to commit application files
  • Maven to build, package, and run the application
  • Docker to containerize the app into a single platform independent image

Usage

  • Jetty

    To build and run the app issue the following from the host command line: mvn clean install

    By default, Spark listens on port 4567, accessible through the browswer via: http://your_host_name_or_ip_address:4567

    If using a model that defines containers for microservices, goFramework will handle getting Spark to listen on the port assigned to the container within the model.

  • Docker

    Build:
    Issue the following from the root directory of the app generation location:

    docker build -t namespace/repository:version .
    namespace - your_company_name
    repository - your_app_name
    version - latest

    The '.' at the end of the build command indicates to use the Dockerfile and content of the current directory

    Run:The following usage of docker run assumes MySQL is in use by the app and is listening on port 3306

    docker run -it -d -p 52000:3306 -p 8080:8080 namespace/repository:version

    52000 is used by way of example for the purpose of avoiding a port conflict in the event MySQL is being run locally on the host machine.
